# AI面试器 - 需求规格

## 用户流程
注册登录 - 选择面试岗位、难度、风格 - 点击开始面试 - 上传简历 - ai面试官提问 - 用户回复 - ai追问（可选，3轮以内） - 重复前面三个环节 - 面试结束，生成评价 - 切换界面查看所有记录和评价分析

## 功能需求

### 用户管理模块
- 用户注册与登录

依赖: 无
对外接口: 无

### 面试功能模块
- 上传简历: 可从本地或个人简历库上传简历给本次面试，上传的简历存入个人简历库，上传后方可发起面试
- 发起面试
- 历史面试记录查询: 可排序、删除、关键词搜索面试会话

依赖: 从面试记录与评价模块获取历史面试会话
对外接口: 无

### 个人能力分析模块
- 个人能力分析界面: 综合历史面试记录，提供的不只有能力分析、面试能力趋势图，还有人生发展建议

依赖: 从面试记录与评价模块获取历史面试会话
对外接口: 无

### 面试通信模块
- 文字交流 
- 语音通话: 后续扩展
- 视频通话: 后续扩展，默认方式，不可用则降级为文字和语音
- 共享界面: 后续扩展，用于一方展示、一方观看同一个屏幕

依赖: 无
对外接口: 提供文字记录给面试记录与评价模块

### ai面试官模块
- 简历解析 
- 选择面试岗位、风格与难度
- 提问: 传输上下文（包括简历、岗位、风格、难度）和用户回答给大模型的对话并返回大模型的提问给用户，且分为0式提问（基于新话题）和1式提问（在0式基础上追问，次数不超过3次）。后续考虑构建并使用专门面试问题数据库

依赖: 从面试通信提供的文字获取用户发言
对外接口: 提供面试问题给面试通信模块

### 面试记录与评价模块
- 记录对话: 记录原文并打上时间点
- 视频回放: 后续扩展，记录用户屏幕，需用户手动开启
- 面试评价: 先生成总体面试评价与改进建议，再给出用户最需要改进的3条问答，返回给用户（后续该功能可以考虑给到ai面试官模块增量生成评价）

依赖: 面试通信的文字记录功能
对外接口: 提供对话记录与面试评价给面试功能模块和个人能力分析模块

## 非功能需求
- 并发量 支持50-100人同时面试
- 响应延迟 面试中允许思考延迟在5s以内，但要求流式输出
- 数据存储 面试记录根据用户选择，范围从不保留到保留3个月

## 边界条件和异常处理
- 如果大模型调用超时，选择重试3次，均失败则重连，仍失败则提醒用户要么死等要么结束面试
- 如果用户掉线，只保留成功发出的消息，并在用户上线后让其选择是否重连
- 简历在上传时限制格式，目前只支持图片和pdf
