# task-010: 实现 AI 面试官提问逻辑

## 描述
实现面试官的 prompt 构建和提问策略。

## 核心概念

- **0 式提问**：开启一个新话题时的首次提问，基于简历和岗位独立出题
- **1 式追问**：基于候选人上一回答的追问，深入挖掘当前话题
- **话题**：一个独立的考察主题（如"Redis 缓存一致性"、"系统设计能力"）
- 每个话题内：1 次 0 式提问 + 最多 3 轮 1 式追问 = 最多 4 次提问
- 追问满 3 轮后，必须切换到新话题（重新 0 式提问）

## 具体内容
- 实现面试官 prompt 构建服务（简历+岗位+风格+难度+对话历史）
- 写 prompt 构建测试用例（验证 prompt 包含必要信息）
- 实现 0 式提问逻辑（基于新话题出题，不依赖上一回答）
- 实现 1 式追问逻辑（基于上一回答追问，识别当前话题的追问轮数）
- 追问计数以话题为单位：切换话题后追问计数重置为 0
- 写提问逻辑测试用例（0 式出新话题 / 1 式在 3 轮内追问 / 超 3 轮切换话题）
- 跑测试确认通过

## 验收标准
- pytest 测试通过
- prompt 包含简历、岗位、历史对话等上下文
- 每个话题最多追问 3 轮，之后必须切换新话题
